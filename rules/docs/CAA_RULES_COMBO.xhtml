File Metadata

 

Note

Used to describe or override the default syntax for the data file itself.

     

Global metadata

 

Note

Describes the file contents collectively.

     

Variable metadata

 

Note

Describes each data product individually

     

File Metadata

Order

 

This should appear before global or variable metadata clocks as they affect the reading of subsequent input.

     

Variable metadata

Order

 

Variable metadata blocks must appear in the same order as the variables appear in the data records.

     

File Metadata

Param syntax

 

FILE_NAME

Required

Specifies the name of the file.  Should not include the path, but does include the file type extension.

   

File Metadata

Param syntax

 

FILE_FORMAT_VERSION

Required

CEF Specivication version, as a text string in syntax “CEF-n.m” to identify the cef version.

In this case, “CEF-2.0”

  

File Metadata

Param syntax

 

END_OF_RECORD_MARKER

Required if data records are broken across lines.

This specifies the character to be used to inducate the end of each data record.  The default value is the new line char (\n).  Setting a different character allows long records to be split across lines with new line characters for human readibility.

On read all new line and CR characters, together with white space (including tabs) and the EOR marker are stripped after the record is read.  Neither the exclamation mark nor ampersand may be used an end of record marker.

  
         

Global Metadata

Param syntax

 

Block of metadata begins with tag START_META and ends with END_META parameters.  The value associated with these parameters is the name of the global attribute.

  

START_META = name

 

END_META = name

  

Global Metadata

Param syntax

 

Global metadata block contains the following entries:

ENTRY

VALUE_TYPE

     

Global Metadata

Param syntax

 

ENTRY = the value associated with this entry is the next entry in the global attribute.  Space characters are valud within a text entry, and text entries must be enclosed in double quotes.

     

Global Metadata

Param syntax

 

ENTRY This parameter may be repeated, with each successive value providing the next entry in the attribute.

     

Global Metadata

Param syntax

 

VALUE_TYPE of a global attribute may be used to convert the ascii text entry in each ENTRY into the appropriate type of the value in the data structure.

     

Global Metadata

Param syntax

 

VALUE_TYPE The default value for each global metadata block independently is text.

     

Global Metadata

Param syntax

 

VALUE_TYPE allows changing of type for each subsequent ENTRY within a block until reset with another VALUE_TYPE parameter, or the end of the attribute block is reached.

     

Global Metadata

Param syntax

ISO_TIME

FLOAT

DOUBLE

INT

CHAR

BYTE

Data

Format

 

Data records must be immediately preceded by a line which identifies the end of the data.

 

DS-QMW-TN-0010 Page 3

This takes the form DATA_UNTIL = yyy where www is a quoted string that will be found at the beginning of a line folowing the last data record in the file, or take the valye EOF (not in quotes) if the data is to be read until the end of the file is reached.  Spaces, tables and non-printing characters a are not permitted within the string yyy.

  

Data

Format

 

All data files are record oriented and homogenous – they have a sequence of records each with the same variables in the same order.  Variables that are multi-dimensional take the natural C ordering, and one entry is required for each element.

     

Data

Format

 

In time series data the records are ordered on the monotonic increasing time ariable.

     

Data

Format

 

Each record is ended by a record_delimiter which by default is the new line character (\n).  The record delimiter may not be the comment marker (!) or ampersand (&) or a non-printing character.

     

Data

Format

 

Entries in a record are comma separated and white space surrounding delimiters are ignored

     

Data

Format

 

The record delimiter and all carriage return (\r), new line ('n') and white space characters at the start and end of records and surrounding delimiters are to be ignored

     

Metadata

Format

 

In order to protect white space and commas in text data, this data type is enclosed within double quotes “ “

     

Metadata

Format

 

Quoted text is used when the text itself is to be used as the value of a parameter

     
         
         

Data

Time

 

Time series data must have a time entry associated with each record.

 

Each other entry in the record is associated with that time.

   

Data

Time

 

If present, the time tag must be the first entry in a record.

     

Data

Time

 

By default, time tags are centred within the sampling interval, and the spacing between timetags is the same as that sampling interval

   

Req?

 

Data

Time

 

For other behaviour, use DELTA_PLUS and Delta_MINUS metadata in seconds to describe the sampling or integration interval corresponding to the data and the location of the time tag within that interval.

     

Data

Time

Note

The use of a second variable holding an absolute time is possible, and other variables in a record may also themselves be time variables.

     

Data

Time

 

Time is represented as a text string in the strict ISO format.  This is of the form: yyyy-mm-ddTHH:MM:ss.wwwZ

  

eg 2001-06-06T12:00:00.000Z

  

Data

Data series

 

Data series have no time entry in a record.  If more than one data series is written to the same file, then each data series must contain the same number of records and any one-to-one dependency between variables in a record must be explicitly explained within the file metadata.
